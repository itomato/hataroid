# Atari ST Games Database
# -----------------------
#
# This little database exists to map ST Disk's to their contents; you can map
# using either group+disk-number, or a varient of the CRC32 algorithym. Each
# disk may have several CRC32's associated to it, due to variations in disk
# formats, etc. (MSA, ST, fiddled with, differetn track counts, etc etc.)
#
# This list is directly descended from Bruno Padinha's list, merged with
# other sources. Small portions of this data came from TOSEC database;
#
# Bruno Padinha's list:  http://stgameslist.atari.org
# TOSEC database:        http://www.tosec.org
#
# Much much thanks to everyone from gp32x.com, gp32emu.com, IRC, et al
# who contributed to this listing!
#
# Much of this is gleaned from a wide variety of sources across the internet,
# and we've got a manual labour project whereby people enter new data into this
# listing, in hopes of making it more complete.
#
# See the Revision list below to find sources of updates.
#
# License
# -------
#
# This database may be used by anyone, anytime, free of charge, without
# limits. It would be ideal if additions are submitted to support@codejedi.com
# for addition to the canonicle list.

# Nomenclature
# ------------
#
# Use a very short name for the games; the goal is to work equally well for
# small handhelds and beefy desktops. If the game exists in multiple media
# or you are unsure, include an abbreviation of the author.
#
# Ex:
# OR:1:12345678:Dungeon Master (FTL)
# AU:97:637121234:Dungeon Master (FTL); OIDs (FTL)
#
# Format of Entries
# -----------------
#
# "#" denotes a comment; comments should only occur at the very top of the
#   document, and should not include CRC numbers in them (so as to avoid
#   accidental matches!)
#
# The layout is:
# Group shortname, disk number, CRC-list, game-list
#
# Group shortname is two letter acronym from the list above. If you have a
#   new groupname, let me know (support@codejedi.com), and you can make a
#   quick suggestion.
# Disk number is free form; "1" for orginal disks (2 for the second disk
#   in multi disk original set, of course). No zero padding.
#   You may include "A" or "B" as needed. "398A" is okay.
#   An unnumbered crack should be "0".
# CRC-list: May be empty if no-CRC is known, though this isn't really
#   useful. Please include at least one CRC for each disk image.
# Game-list: The name of the game on the disk. Optionally include the
#   game author/distributor in parenthesis, if useful, and very short.
#   Keep the name short. Separate multiple names with "; " (including
#   both the semicolon and the space!)
#
# Rules for Submission of Updates:
# --------------------------------
# Make sure your group names match the legend.
# Make sure you are not duplicating someone elses work.
# Look for the existing entry first; this database is fairly complete in
#   its compilation disk listing, though many CRC's are missing. If you
#   can just add in a CRC, let me know!
# It is correct to have different CRC's for the same disk -- the format
#   may have changed.. from .ST to .MSA for example, or the track/sector
#   count of the disk.. and still be the same content! So multiple
#   CRC's are fine.
# Email updates to support@codejedi.com -- include only the new material
#   and not the existing material.
# Ideally, notify support@codejedi.com that you'll be tackling some
#   series of disks, so that we can avoid duplication of effort!
# Disk number is not zero padded; "2" and not "02" or "00002" :)
#
